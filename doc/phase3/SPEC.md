# Phase 3: 통합 모니터링 대시보드

## 목적
"한눈에 보기 쉽게 확인하고 관리"

## 기능 요구사항

### 3-1. 프로젝트 개요 페이지 강화
- 서비스 건강 상태 카드 그리드 (서비스명, 상태 dot, 응답 시간)
- 요약: 정상/오류/미확인 카운트
- "전체 검증" 버튼 + 프로그레스 표시

### 3-2. 상태 모니터링 전용 페이지
- `/project/[id]/health` 경로
- 필터 (전체/정상/오류)
- 서비스별 상세 펼침: 검증 이력 타임라인 + 응답시간 스파크라인

### 3-3. 서비스맵 건강 상태 연동
- 서비스 디테일 시트에 "검증 실행" 버튼 + 최근 5건 이력
- 필수 env var 개수 표시

### 3-4. 컴포넌트
- health-summary-card: 개별 서비스 상태 카드
- health-timeline: 검증 이력 타임라인
- health-sparkline: CSS 기반 응답시간 추이 차트
