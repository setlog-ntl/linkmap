## 평가 기준

템플릿 유지/교체 판단 시 고려 사항:

1. **GitHub Pages 호환성**: 정적 빌드 (`next export`) 가능한가?
2. **사용자 수요**: 카테고리별 균형 (개인, 비즈니스, 크리에이터)
3. **커스터마이징 용이성**: env var만으로 기본 설정 변경 가능한가?
4. **유지보수 부담**: 외부 의존성 최소화, 보안 취약점 관리
5. **차별화**: Linkmap 플랫폼만의 가치를 제공하는가?

## 향후 템플릿 후보 카테고리

### 우선순위 높음
- **이벤트/웨딩 페이지**: 일회성 이벤트 초대장, RSVP
- **레스토랑/카페 메뉴**: QR 코드 연동 디지털 메뉴
- **포토 갤러리**: 사진작가/아티스트용 포트폴리오

### 우선순위 중간
- **이력서/CV**: 인쇄 가능한 온라인 이력서
- **제품 랜딩**: 단일 제품 소개 + CTA
- **커뮤니티/팬페이지**: 팬덤/동호회 허브

### 우선순위 낮음
- **문서 사이트**: 기술 문서/가이드 (경쟁 많음)
- **뉴스레터 랜딩**: 구독 양식 + 아카이브

## 번들 구조 개선 방향

### 현재 구조
템플릿은 GitHub 레포를 통째로 fork하는 방식. 모든 파일이 하나의 레포에 존재.

### 개선 방향
1. **핵심 파일 분리**: `index.html`, `styles.css`, `config.json` 등 사용자가 편집할 파일만 DB에 저장
2. **프레임워크 공통 베이스**: 공통 빌드 설정을 base repo에 두고, 템플릿은 콘텐츠 파일만 관리
3. **프리뷰 시스템**: 각 템플릿에 정적 프리뷰 이미지 + 라이브 프리뷰 URL 추가

### 프리뷰 이미지 계획
- `preview_image_url` 컬럼 활용 (현재 모두 null)
- GitHub Pages로 배포된 데모 사이트 스크린샷 자동 생성
- OG 이미지 용도로도 활용 (1200x630)
