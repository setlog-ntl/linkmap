# 원클릭 배포 템플릿 확장 기획 레퍼런스

> **Linkmap** 원클릭 홈페이지 배포 기능의 템플릿을 5개 → 20개로 확장하기 위한 종합 기획서.
> 시장 리서치, 심리학적 분석, 페르소나 정의, 우선순위 매트릭스, 구현 가이드를 포함한다.

---

## 1. AI 구현용 프롬프트

아래 프롬프트를 AI(Claude Code, Cursor 등)에 전달하면 이 기획서를 기반으로 템플릿을 구현할 수 있다.

```
## 컨텍스트
Linkmap(https://linkmap.vercel.app)은 바이브 코딩을 위한 서비스 연결 & API 관리 플랫폼이다.
원클릭 배포 기능(/oneclick)으로 사용자가 GitHub 연결 → 템플릿 선택 → Vercel 배포 3단계만으로 개인 홈페이지를 생성한다.

## 기술 스택
- Next.js 16 (App Router) + TypeScript + Tailwind CSS v4
- Vercel 배포, GitHub Fork 기반
- 반응형 디자인 (모바일 퍼스트)
- 다크모드 지원 (next-themes)

## 작업 요청
docs/레퍼런스.md의 "추천 템플릿 리스트 (20개)" 섹션을 참조하여 다음 템플릿을 구현하라:

### 템플릿: [템플릿명]
- 타겟: [페르소나]
- 카테고리: [링크인바이오 | 전문가 | 비즈니스 | 커뮤니티]
- 핵심 섹션: [섹션 목록]
- 디자인 테마: [디자인 특징]

### 요구사항
1. `linkmap-templates/[slug]` GitHub 레포에 Next.js 프로젝트 생성
2. 모든 개인화 데이터는 `NEXT_PUBLIC_*` 환경변수로 주입
3. Lighthouse 성능 90+ / 접근성 90+
4. 한국어 기본, 영어 전환 지원
5. OG 이미지 자동 생성 (/api/og)
6. 반응형: 모바일(360px) ~ 데스크톱(1440px)
7. 다크모드 토글 포함
8. Google Analytics 4 연동 (NEXT_PUBLIC_GA_ID)
9. SEO 메타태그 + JSON-LD 구조화 데이터
10. `src/data/homepage-templates.ts`에 시드 데이터 추가
11. `homepage_templates` 테이블에 INSERT 마이그레이션 작성

### 환경변수 규칙
- 필수: NEXT_PUBLIC_SITE_NAME
- 선택: NEXT_PUBLIC_* (사이트별 커스터마이징)
- 비공개: GA_ID, API 키 등은 Vercel 환경변수로 관리

### 디자인 원칙
- 미니멀리즘: 불필요한 장식 없이 콘텐츠 중심
- 타이포그래피: Pretendard(한글) + Inter(영문)
- 컬러: CSS 변수 기반 테마 커스터마이징
- 애니메이션: 로딩 시 subtle fade-in, scroll 트리거 (framer-motion)
- 아이콘: Lucide Icons 통일
```

## 2. 심리학적 분석

### 2.1 개인 웹사이트를 만드는 6가지 핵심 동기

| # | 동기 | 심리학적 근거 | 설계 반영 |
|---|------|--------------|----------|
| 1 | **자기표현 (Self-Expression)** | Maslow 자아실현, 자기결정이론(SDT) | 고도의 커스터마이징, 테마 선택권 |
| 2 | **사회적 승인 (Social Validation)** | 사회적 비교 이론, SNS 좋아요 문화 | 공유 버튼, OG 프리뷰, 방문자 카운터 |
| 3 | **통제감 (Sense of Control)** | 자기효능감, "나만의 공간" 소유 | 내 도메인, 내 코드, Fork 기반 완전 소유 |
| 4 | **진정성 (Authenticity)** | MZ세대 유동적 정체성, 인지부조화 감소 | 스토리텔링 섹션, 가치관 표현 공간 |
| 5 | **효율성 (Efficiency)** | 인지 부하 감소, 정보 접근성 | 원클릭 배포, 템플릿 자동화 |
| 6 | **소속감 (Belonging)** | 커뮤니티 동일시, 직업적 정체성 | 직군별 템플릿, 커뮤니티 쇼케이스 |

### 2.2 페르소나별 핵심 Pain Point

| 페르소나 | 핵심 감정 | Pain Point | 해결 전략 |
|---------|----------|-----------|----------|
| 신입 개발자 | 불안감, 자기증명 | "포트폴리오가 없으면 취업 못 함" | GitHub 연동 쇼케이스 |
| 디자이너 | 완벽주의, 표현욕 | "내 디자인 감각을 보여줘야 함" | 비주얼 중심 갤러리 |
| 프리랜서 | 신뢰 구축 | "전문성을 증명할 웹 존재감 필요" | 경력+후기+연락처 통합 |
| 콘텐츠 창작자 | 플랫폼 의존 불안 | "유튜브/인스타가 사라지면?" | 자체 링크 허브 |
| 취준생 | 차별화 갈증 | "PDF 이력서로는 어필 안 됨" | 인터랙티브 이력서 |
| 소상공인 | 디지털 소외감 | "홈페이지 만들 줄 모름" | 초간단 템플릿 |
| 스터디 운영자 | 정보 분산 | "노션/카톡/구글폼 흩어짐" | 통합 허브 페이지 |
| 연구자 | 학술적 권위 | "논문/강의 정리 필요" | 학술 포트폴리오 |
| 사이드 프로젝트 런처 | 검증 욕구 | "아이디어를 빠르게 런칭" | 제품 랜딩페이지 |
| 비영리 단체 | 투명성 | "활동을 알려야 후원받음" | 미션+활동+후원 통합 |
| 글쓰기 커뮤니티 | 창작 욕구 | "내 글을 모아 보여주고 싶음" | 블로그/매거진 |
| 이벤트 기획자 | 참여 유도 | "이벤트 정보를 한곳에" | 카운트다운+신청폼 |

### 2.3 디지털 정체성 효과

MZ세대 디지털 정체성 트렌드 (2025-2026):

- **셀프 브랜딩 일상화**: 개인 브랜딩이 선택이 아닌 필수. IT, 마케팅, 디자인 등 전문분야에서 커리어 SNS를 통한 포트폴리오 공유 확산
- **디지털 아바타 부상**: 가상의 디지털 정체성을 창조하고 활용하는 트렌드 강화, 2025년 디지털 아바타 채택률 70% 수준 예측
- **진정성 추구**: Z세대는 과도한 상업성보다 진정성 있는 콘텐츠에 끌림. 꾸며진 이미지보다 본인의 진정한 가치와 철학을 전달하는 방향
- **자기이해 기반 의사결정**: 성향 테스트, 심리 분석 등을 활용한 자기이해 콘텐츠 소비 → 소비와 생활 선택으로 확대
- **가치 소비와 윤리적 브랜딩**: 개인 브랜딩을 통해 가치관과 사회적 책임을 표현

> **시사점**: 템플릿 설계 시 "나다움"을 표현할 수 있는 커스터마이징 자유도와 진정성 있는 스토리텔링 공간이 핵심 경쟁력.

## 7. 트렌드 기반 인사이트

### 7.1 한국 시장 특수성

**링크인바이오 시장 급성장**
- Linktree가 글로벌 시장 지배하나 한국에서는 한국어 기반 서비스가 강세
- 주요 국내 서비스: **LinkgraP**(링크그래피), **Lit.link**(리틀리), **CDBD**, **Wlinky**
- LinkgraP: 올인원 퍼스널 브랜딩 플랫폼 (예약, 설문, 뉴스레터 통합)
- 2025년 기준 링크인바이오 서비스 종류 기하급수적 증가

**모바일 쇼핑 시장 통계 (2025)**
- 2025.8월 온라인쇼핑 거래액: **22조 4,802억원** (전년대비 +6.6%)
- 모바일쇼핑 거래액: **17조 8,489억원** (전년대비 +8.9%)
- 모바일쇼핑 비중: **79.4%** (전년 77.7%에서 +1.7%p)
- → **모바일 퍼스트 디자인이 절대적으로 중요**

**소상공인 디지털 전환**
- 전체 온라인 거래의 약 77~79%가 모바일 경유
- 정부 차원의 소상공인 디지털 전환 지원 정책 확대
- 소상공인 소매업체의 온라인 진출 가속화

**MZ세대 디지털 정체성**
- 셀프 브랜딩 일상화: 개인 브랜딩이 "선택 → 필수"로 전환
- 진정성 추구: 꾸며진 이미지보다 본인의 가치와 철학 전달 선호
- 자기이해 기반 소비: 성향 테스트, MBTI 등 자기이해 콘텐츠 → 소비 결정으로 확대

### 7.2 글로벌 트렌드

**노코드/로우코드 시장**
- 글로벌 시장 규모: 2025년 **455억 달러** (약 59조원)
- 노코드 AI 플랫폼: 2025년 65.6억 달러, 연평균 31.13% 성장 전망
- Framer: 노코드 웹빌더 대표주자로 **20억 달러 기업가치** 달성 (2025년 1억 달러 투자 유치)
- 비전문가(디자이너, 마케터)의 웹사이트 제작 참여 본격화

**Next.js + Vercel 생태계 (2025-2026)**
- 모든 신규 템플릿이 App Router 지원, Pages Router 점진 감소
- 기본 스택 표준화: **App Router + TypeScript + Tailwind CSS**
- 용도별 템플릿 분화: SaaS, 블로그, 대시보드, 이커머스
- Vercel 원클릭 배포 + Edge 캐싱 + 이미지 자동 최적화
- Next.js 16: Turbopack 안정화, 명시적 캐싱, 개발자 도구 개선

**웹 디자인 트렌드**
- 네온/맥시멀리즘 & Apple Aesthetic 양극화
- 브루탈리즘 디자인의 부활
- AI 생성 그래픽/일러스트레이션 활용
- 마이크로 인터랙션 & 스크롤 트리거 애니메이션

## 8. 경쟁 분석

### 8.1 경쟁사 비교

| 경쟁사 | 유형 | 강점 | 약점 | Linkmap 차별화 |
|--------|------|------|------|---------------|
| **Linktree** | 링크인바이오 | 글로벌 1위, 간편한 UX | 커스터마이징 제한, 코드 소유권 없음 | Fork 기반 코드 완전 소유 |
| **LinkgraP** | 링크인바이오 (한국) | 한국어, 올인원 기능 | 커스텀 코드 불가, 플랫폼 종속 | 코드 소유 + 서비스 관리 통합 |
| **Wix** | 웹사이트 빌더 | WYSIWYG 에디터, 풍부한 템플릿 | 무겁고 느림, 코드 이동 불가 | 가볍고 빠른 Next.js, 코드 이동 자유 |
| **Notion + Oopy** | 노코드 웹 | 쉬운 콘텐츠 관리 | 디자인 자유도 낮음, SEO 약함 | 완전한 디자인 자유도, SEO 최적화 |
| **Vercel Templates** | 개발자 템플릿 | 최고 품질, 공식 지원 | 개발자 전용, 비개발자 접근 어려움 | 비개발자도 3단계로 배포 |
| **Framer** | 노코드 웹빌더 | 디자인 품질, 인터랙션 | 유료(월 $15+), 코드 소유 불가 | 무료 + 코드 소유, 개발자 확장 가능 |

### 8.2 Linkmap 차별화 전략 (5가지)

1. **코드 소유권**: Fork 기반으로 사용자가 코드를 완전히 소유. 플랫폼 종속 없음
2. **서비스 관리 통합**: 홈페이지 + API 키/환경변수/서비스 연결을 한곳에서 관리
3. **개발자 확장성**: 비개발자는 템플릿 그대로 사용, 개발자는 코드 수정으로 무한 확장
4. **무료 배포**: Vercel 무료 플랜으로 커스텀 도메인까지 무료
5. **한국어 우선**: 한국 시장에 최적화된 템플릿과 가이드 (카카오맵, 네이버 SEO 등)

## 9. 각 템플릿 상세 설명

### 9.1 링크인바이오 프로 (`link-in-bio-pro`)

**타겟**: 인스타그램/유튜브/틱톡 크리에이터, 인플루언서
**핵심 섹션**:
- 프로필 사진 + 바이오 (애니메이션 배경)
- 링크 버튼 리스트 (아이콘 + 제목 + URL)
- 소셜 미디어 아이콘 바
- 최신 콘텐츠 임베드 (유튜브/인스타)
- 테마 선택 (10+ 프리셋 테마)

**디자인 특징**:
- 모바일 중심 (360px 최적화)
- 풀스크린 단일 페이지
- 그라데이션/글래스모피즘 배경 옵션
- 부드러운 hover 애니메이션

**심리적 효과**: 자기표현 + 사회적 승인 (SNS 프로필에 링크 게시 → 방문 유도)
**예상 유입 효과**: 인스타그램 바이오 링크 → 높은 바이럴 가능성

**환경변수**:
```
NEXT_PUBLIC_SITE_NAME     # 사이트 이름 (필수)
NEXT_PUBLIC_BIO           # 소개 문구
NEXT_PUBLIC_AVATAR_URL    # 프로필 이미지 URL
NEXT_PUBLIC_THEME         # 테마 프리셋 (neon/minimal/gradient/...)
NEXT_PUBLIC_GA_ID         # Google Analytics
```

### 9.2 디지털 명함 (`digital-namecard`)

**타겟**: 프리랜서, 직장인, 자영업자
**핵심 섹션**:
- 이름 + 직함 + 회사
- 연락처 (전화, 이메일, 주소)
- 소셜 미디어 링크
- QR 코드 (vCard 형식 자동 생성)
- NFC 탭 안내 (선택)
- "연락처에 저장" 버튼

**디자인 특징**:
- 실물 명함 느낌의 카드 레이아웃
- 미니멀한 타이포그래피 중심
- 다크/라이트 모드에서 모두 세련된 느낌
- 인쇄 가능한 CSS (명함 크기 맞춤)

**심리적 효과**: 통제감 + 효율성 (종이 명함 → 디지털 전환, 언제든 업데이트)
**예상 유입 효과**: 오프라인 미팅 후 QR 스캔 → 웹 방문 → 자연 바이럴

**환경변수**:
```
NEXT_PUBLIC_SITE_NAME     # 이름 (필수)
NEXT_PUBLIC_TITLE         # 직함
NEXT_PUBLIC_COMPANY       # 회사명
NEXT_PUBLIC_EMAIL         # 이메일
NEXT_PUBLIC_PHONE         # 전화번호
NEXT_PUBLIC_ADDRESS       # 주소
```

### 9.3 개발자 쇼케이스 (`dev-showcase`)

**타겟**: 신입/주니어 개발자, 취준생
**핵심 섹션**:
- 히어로: 이름 + 한줄 소개 + 타이핑 애니메이션
- About: 자기소개, 기술 스택 아이콘 그리드
- Projects: GitHub 핀 프로젝트 카드 (자동 연동)
- Experience: 타임라인 형태 경력/교육
- Blog: 최근 글 목록 (RSS or 마크다운)
- Contact: 이메일 + GitHub + LinkedIn

**디자인 특징**:
- 다크 테마 기본 (개발자 친화적)
- 터미널/코드 에디터 느낌의 UI 요소
- GitHub 잔디밭(Contribution Graph) 임베드
- 스크롤 트리거 애니메이션

**심리적 효과**: 자기증명 + 소속감 (개발자 커뮤니티 정체성)
**예상 유입 효과**: GitHub README에 링크 → 채용담당자 유입

**환경변수**:
```
NEXT_PUBLIC_SITE_NAME       # 이름 (필수)
NEXT_PUBLIC_GITHUB_USERNAME # GitHub 사용자명
NEXT_PUBLIC_TAGLINE         # 한줄 소개
NEXT_PUBLIC_SKILLS          # 기술 스택 (JSON)
NEXT_PUBLIC_RESUME_URL      # 이력서 PDF URL
```

### 9.4 소상공인 비즈니스 (`small-biz`)

**타겟**: 카페, 음식점, 미용실, 소매점 사장님
**핵심 섹션**:
- 가게 이름 + 대표 이미지
- 메뉴/서비스 목록 (가격 포함)
- 영업시간
- 위치 (카카오맵 임베드)
- 전화 연결 버튼 (모바일 `tel:` 링크)
- 리뷰/별점 표시 (선택)

// components/DevHero.tsx
'use client';

import { useEffect, useState } from 'react';
import { Github, Linkedin, Mail } from 'lucide-react';

const TYPING_TEXTS = [
  'Frontend Developer',
  'React Enthusiast',
  'Open Source Contributor',
];

export default function DevHero() {
  const name = process.env.NEXT_PUBLIC_SITE_NAME || 'Kim Developer';
  const github = process.env.NEXT_PUBLIC_GITHUB_USERNAME || 'octocat';
  const [textIndex, setTextIndex] = useState(0);
  const [charIndex, setCharIndex] = useState(0);
  const [isDeleting, setIsDeleting] = useState(false);

  useEffect(() => {
    const currentText = TYPING_TEXTS[textIndex];
    const timeout = setTimeout(() => {
      if (!isDeleting) {
        if (charIndex < currentText.length) {
          setCharIndex(charIndex + 1);
        } else {
          setTimeout(() => setIsDeleting(true), 2000);
        }
      } else {
        if (charIndex > 0) {
          setCharIndex(charIndex - 1);
        } else {
          setIsDeleting(false);
          setTextIndex((textIndex + 1) % TYPING_TEXTS.length);
        }
      }
    }, isDeleting ? 50 : 100);
