# 원클릭 배포 템플릿 마스터 기획서 (트렌드·바이럴 기반 재설계)

> **Linkmap** 원클릭 홈페이지 배포 기능의 15개 템플릿을 4개 Phase로 나누어 구현하기 위한 마스터 기획서.
> 트렌드·바이럴 관점으로 전면 재설계 (2026-02). 각 템플릿별 상세 기획은 하위 폴더의 개별 문서를 참조한다.

---

## 1. CTO 의사결정 요약

### 왜 이 15개 템플릿과 Phase인가?

링크인바이오 시장 CAGR 13.2% ($1.62억→$4.24억), Linktree 79.95% 점유율 속에서 "코드 소유 + 무료 배포" 대안 기회가 명확하다. 한국 신규 창업자 70%가 온라인 기반이며, QR코드 스캔이 4년간 433% 증가해 디지털 명함·QR메뉴 수요가 폭발 중이다. 크리에이터 95%가 Direct-to-Fan 전환 중이어서, 플랫폼 종속 없는 개인 허브 페이지의 가치가 사상 최고다.

Phase 1은 SNS 바이럴 계수가 가장 높은 3개(링크인바이오, 디지털 명함, 개발자 쇼케이스)로 유지하되, Phase 2~4를 트렌드 데이터 기반으로 재배치하고, 기존 15개 중 3개를 통합/제거하며 신규 3개를 추가해 총 15개를 유지한다.

**핵심 변경 근거**:
1. **creator-hub → link-in-bio-pro 통합**: 기능 93% 중복, D2F 트렌드에 맞춰 "크리에이터 모드"로 흡수
2. **creative-portfolio → dev-showcase 통합**: 단독 니치 좁음, "디자이너 모드"로 통합이 관리·개발 비용 효율적
3. **academic-page 제거**: 바이럴 점수 최하위(70점), 타겟 너무 좁음

빈 슬롯에 **QR메뉴·뉴스레터·스터디모집**을 배치해 소상공인 B2B·D2F·커뮤니티 바이럴을 강화한다.

### Phase 분할 근거

우선순위 매트릭스: **시장수요(40%) + 바이럴가능성(30%) + 수익화잠재력(20%) + 구현용이도(10%)**

| Phase | 기간 | 템플릿 수 | 전략 |
|-------|------|-----------|------|
| Phase 1: 바이럴·DAU 최우선 | 2주 | 3개 | 최고 바이럴 계수 → DAU 100+, 배포 500개 |
| Phase 2: 수익화·성장 | 3주 | 5개 | B2B+Pro 전환 → Pro 전환율 5%+, 다운로드 2,000+ |
| Phase 3: 확장·니치 | 3주 | 4개 | 전문가 시장 침투 → 프리랜서 플랫폼 연계 |
| Phase 4: 커뮤니티·롱테일 | 2주 | 3개 | 커뮤니티 채널 확보 → 전체 15개 달성 |

## 8. 공통 기술 규격

### 8.1 기술 스택

| 항목 | 스택 |
|------|------|
| Framework | Next.js 16 (App Router) |
| Language | TypeScript (strict mode) |
| Styling | Tailwind CSS v4 |
| Fonts | Pretendard (한글) + Inter (영문) via `next/font` |
| Icons | Lucide React |
| Animation | framer-motion (선택) |
| Dark Mode | next-themes |
| SEO | next/metadata + JSON-LD |
| OG Image | @vercel/og (`/api/og`) |
| Analytics | Google Analytics 4 (선택) |
| Deploy | GitHub Pages (GitHub Fork 기반) |

### 8.2 레포 구조

모든 템플릿은 `linkmap-templates/{slug}/` 표준 구조를 따른다:

```
linkmap-templates/{slug}/
├── public/
│   ├── favicon.ico
│   └── og-image.png
├── src/
│   ├── app/
│   │   ├── layout.tsx         # 메타데이터, 폰트, 테마
│   │   ├── page.tsx           # 메인 페이지
│   │   └── api/og/route.tsx   # OG 이미지 생성
│   ├── components/            # 섹션별 컴포넌트
│   └── lib/
│       └── config.ts          # 환경변수 → 타입 안전 config
├── tailwind.config.ts
├── next.config.ts
├── package.json
├── tsconfig.json
└── README.md                  # 환경변수 설명, 커스터마이징 가이드
```

### 8.3 환경변수 규칙

- **필수**: `NEXT_PUBLIC_SITE_NAME` (모든 템플릿 공통)
- **선택**: `NEXT_PUBLIC_*` 패턴 (사이트별 커스터마이징)
- **비공개**: `GA_ID`, API 키 등은 Vercel/GitHub Secrets으로 관리
- **기본값**: 환경변수 미설정 시에도 의미 있는 데모가 표시되어야 함

### 8.4 성능 기준

| 항목 | 목표 |
|------|------|
| Lighthouse Performance | 90+ |
| Lighthouse Accessibility | 90+ |
| Lighthouse Best Practices | 90+ |
| Lighthouse SEO | 90+ |
| First Contentful Paint | < 1.5s |
| Largest Contentful Paint | < 2.5s |
| Cumulative Layout Shift | < 0.1 |

### 8.5 SEO 요구사항

- `next/metadata` API로 메타태그 생성
- JSON-LD 구조화 데이터 (Person, Organization, WebSite 등)
- `/api/og` 엔드포인트로 동적 OG 이미지 생성
- `robots.txt` + `sitemap.xml` 자동 생성
- 한국어 기본 (`lang="ko"`)

### 8.6 디자인 원칙

1. **모바일 퍼스트**: 360px → 768px → 1024px → 1440px
2. **콘텐츠 중심**: 장식보다 콘텐츠가 돋보이는 디자인
3. **접근성**: WCAG 2.1 AA 준수, 키보드 내비게이션, 스크린리더 호환
4. **일관성**: 모든 템플릿에서 공통 UX 패턴 (다크모드 토글, 푸터, OG 이미지)
5. **타이포그래피**: Pretendard(한글) + Inter(영문), CSS 변수 기반 테마
