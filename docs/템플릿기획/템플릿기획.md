# 원클릭 배포 템플릿 확장 마스터 기획서

> **Linkmap** 원클릭 홈페이지 배포 기능의 15개 신규 템플릿을 4개 Phase로 나누어 구현하기 위한 마스터 기획서.
> 각 템플릿별 상세 기획은 하위 폴더의 개별 문서를 참조한다.

---

## 1. CTO 의사결정 요약

### 왜 15개 템플릿인가?

현재 Linkmap의 원클릭 배포 기능은 2개의 홈페이지 템플릿(`homepage-minimal`, `homepage-links`)만 제공한다.
`docs/레퍼런스.md`의 시장 리서치 결과, 다음 근거로 15개 신규 템플릿 확장을 결정했다:

1. **시장 수요 다변화**: 12개 페르소나 분석 결과, 개발자·디자이너·소상공인·크리에이터·비영리단체 등 타겟별 니즈가 명확히 다름
2. **경쟁 우위 확보**: Linktree(링크 전용), Framer(유료), Wix(무거움) 대비 "코드 소유 + 무료 배포 + 다양한 템플릿"이 핵심 차별화
3. **바이럴 성장 엔진**: 링크인바이오/디지털 명함은 SNS 프로필에 노출되어 자연 바이럴 유입 발생
4. **수익화 기반**: 프리미엄 템플릿 → Pro 플랜 전환 유도, 소상공인 B2B 채널 진입

### Phase 분할 근거

우선순위 매트릭스(시장수요 40% + 바이럴가능성 30% + 수익화잠재력 20% + 구현용이도 10%)를 기반으로:

| Phase | 기간 | 템플릿 수 | 전략 |
|-------|------|-----------|------|
| Phase 1: MVP+ | 2주 | 3개 | 최고 점수 3개 → 빠른 시장 검증, DAU 100+ 달성 |
| Phase 2: 성장 | 3주 | 5개 | 비즈니스 가치 높은 5개 → Pro 전환율 5%+, 소상공인 채널 진출 |
| Phase 3: 확장 | 3주 | 4개 | 전문가/니치 마켓 4개 → 템플릿 포트폴리오 완성 |
| Phase 4: 커뮤니티 | 2주 | 3개 | 커뮤니티/비영리 3개 → 사회적 가치 + 롱테일 |

## 2. 공통 기술 규격

### 2.1 기술 스택

| 항목 | 스택 |
|------|------|
| Framework | Next.js 16 (App Router) |
| Language | TypeScript (strict mode) |
| Styling | Tailwind CSS v4 |
| Fonts | Pretendard (한글) + Inter (영문) via `next/font` |
| Icons | Lucide React |
| Animation | framer-motion (선택) |
| Dark Mode | next-themes |
| SEO | next/metadata + JSON-LD |
| OG Image | @vercel/og (`/api/og`) |
| Analytics | Google Analytics 4 (선택) |
| Deploy | GitHub Pages (GitHub Fork 기반) |

### 2.2 레포 구조

모든 템플릿은 `linkmap-templates/{slug}/` 표준 구조를 따른다:

```
linkmap-templates/{slug}/
├── public/
│   ├── favicon.ico
│   └── og-image.png
├── src/
│   ├── app/
│   │   ├── layout.tsx         # 메타데이터, 폰트, 테마
│   │   ├── page.tsx           # 메인 페이지
│   │   └── api/og/route.tsx   # OG 이미지 생성
│   ├── components/            # 섹션별 컴포넌트
│   └── lib/
│       └── config.ts          # 환경변수 → 타입 안전 config
├── tailwind.config.ts
├── next.config.ts
├── package.json
├── tsconfig.json
└── README.md                  # 환경변수 설명, 커스터마이징 가이드
```

### 2.3 환경변수 규칙

- **필수**: `NEXT_PUBLIC_SITE_NAME` (모든 템플릿 공통)
- **선택**: `NEXT_PUBLIC_*` 패턴 (사이트별 커스터마이징)
- **비공개**: `GA_ID`, API 키 등은 Vercel/GitHub Secrets으로 관리
- **기본값**: 환경변수 미설정 시에도 의미 있는 데모가 표시되어야 함

### 2.4 성능 기준

| 항목 | 목표 |
|------|------|
| Lighthouse Performance | 90+ |
| Lighthouse Accessibility | 90+ |
| Lighthouse Best Practices | 90+ |
| Lighthouse SEO | 90+ |
| First Contentful Paint | < 1.5s |
| Largest Contentful Paint | < 2.5s |
| Cumulative Layout Shift | < 0.1 |

### 2.5 SEO 요구사항

- `next/metadata` API로 메타태그 생성
- JSON-LD 구조화 데이터 (Person, Organization, WebSite 등)
- `/api/og` 엔드포인트로 동적 OG 이미지 생성
- `robots.txt` + `sitemap.xml` 자동 생성
- 한국어 기본 (`lang="ko"`)

### 2.6 디자인 원칙

1. **모바일 퍼스트**: 360px → 768px → 1024px → 1440px
2. **콘텐츠 중심**: 장식보다 콘텐츠가 돋보이는 디자인
3. **접근성**: WCAG 2.1 AA 준수, 키보드 내비게이션, 스크린리더 호환
4. **일관성**: 모든 템플릿에서 공통 UX 패턴 (다크모드 토글, 푸터, OG 이미지)
5. **타이포그래피**: Pretendard(한글) + Inter(영문), CSS 변수 기반 테마
